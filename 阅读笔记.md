# 代码阅读笔记：Pruning as a Cooperative Game (ICLR 2026)

## 1. 核心思想

论文将 LLM 层剪枝问题建模为**合作博弈**：每一层视为一个"玩家"，利用 **Shapley 值** 估计每层对模型整体性能的贡献度，按贡献排序后移除贡献最小的层。

## 2. 整体流程（三步走）

主入口文件 `sharpley_agent_ppl.py` 通过 `step` 变量控制执行哪一步：

| 步骤 | 函数 | 作用 |
|------|------|------|
| **Step 1** | `step1_save_data()` | **采样 + 评估**：使用分层蒙特卡洛生成大量 layer mask，对每个 mask 实际运行模型计算 PPL，得到 `(mask, score)` 训练数据 |
| **Step 2** | `step2_train_model()` | **训练代理网络**：用一个轻量 MLP（`NeuralNet`）拟合 mask → score 的映射关系，作为代理模型 |
| **Step 3** | `step3_estimate_contrib()` | **估计 Shapley 值**：对每一层，用代理网络快速计算"有该层 vs 无该层"的分数差，近似其 Shapley 贡献值 |

## 3. 关键模块详解

### 3.1 分层蒙特卡洛采样 — `stratified_by_hamming_weight()`

- 生成二进制 mask 向量（1=保留层, 0=移除层）
- **按 Hamming weight（保留层数）分层**，确保不同剪枝比例均有覆盖
- Llama-2-7B (32层) 默认采样 `ks=(30, 27, 24, 21, 18)`，对应保留 56%~94% 的层
- Mamba (64层) 使用 `ks=(60, 56, 52, 48, 42, 38, 34)`

### 3.2 高效掩码推理 — `zero_out_layer_params()`

- 使用 **context manager** 模式：临时将被 mask 的层的所有参数置零，推理完后恢复
- 避免反复复制/删除层，大幅节省内存
- 不同模型架构访问层的方式不同：
  - Llama: `model.model.layers[idx]`
  - Mamba: `model.backbone.layers[idx]`
  - RWKV: `model.rwkv.blocks[idx]`

### 3.3 代理网络 — `NeuralNet`

```
Linear(N, 2N) → CELU → Linear(2N, 1) → Sigmoid
```
- 输入：N 维二进制 mask（N=层数）
- 输出：[0,1] 的 score（`baseline_ppl / masked_ppl`，越接近 1 越好）
- 训练：SGD + MSE loss，300 epochs

### 3.4 Shapley 值估计 — `step3_estimate_contrib()`

对每一层 m，通过 80000 次采样近似 Shapley 值：

$$\phi_m = \frac{1}{MC} \sum_{s=1}^{MC} \left[ f(\text{mask}^{(s)} \cup \{m\}) - f(\text{mask}^{(s)} \setminus \{m\}) \right]$$

- 直接用代理网络 `net()` 做前向推理，无需再跑完整大模型
- 贡献值越低的层越适合被剪枝

## 4. 剪枝执行 — `prune/prune.py`

- `delete_layers(model, layer_indices)`: 按 Shapley 排名删除指定层
  - 从后往前 `del` 层（避免索引错位）
  - 重新编号 `layer_idx`
  - 更新 `config.num_hidden_layers`

## 5. 层合并（可选增强）— `utils.py`

提供两种层合并策略，用于替代简单删除：

| 函数 | 策略 |
|------|------|
| `merge_layers_return_model()` | 权重加权融合（`weight_factor` 控制） |
| `merge_layers_with_bias_compensation()` | 融合 + 偏置补偿：给 Linear 层添加 bias，计算原始/合并模型的隐层差异作为补偿 |

## 6. 评估模块 — `evaluate/`

| 文件 | 功能 |
|------|------|
| `eval_ppl.py` | 在 WikiText-2、PTB、C4 上计算 PPL |
| `dataset.py` | 数据加载器（支持滑动窗口分割） |
| `lora_retrain.py` | 剪枝后 LoRA 微调恢复性能 |
| `quantize_gptq.py` | GPTQ 量化支持 |

## 7. 模型适配 — `models/`

| 文件 | 说明 |
|------|------|
| `modeling_dropped_llama.py` | 修改版 Llama 模型（基于 transformers 4.38.1），支持跳层推理 |
| `configuration_dropped_llama.py` | 对应的 config，增加 `dropped_layer_list` 字段 |
| `short_hf.py` | ShortGPT 的 Block Influence 基线实现 |

## 8. 多架构支持

仓库为四种架构提供了独立的入口文件：

| 文件 | 架构 | 层访问路径 |
|------|------|-----------|
| `sharpley_agent_ppl.py` | Llama | `model.model.layers` |
| `sharpley_agent_ppl_mamba.py` | Mamba | `model.backbone.layers` |
| `sharpley_agent_ppl_rwkv.py` | RWKV | `model.rwkv.blocks` |
| `sharpley_agent_ppl_quantize.py` | 量化模型 (GPTQ) | `model.model.layers` |

## 9. 辅助功能

- `metrics.py`: 实现 **Block Influence** 指标（余弦相似度 / 角距离），用于 ShortGPT 基线对比
- `peft/`: 内嵌了一份定制的 PEFT 库，用于剪枝后 LoRA 微调

## 10. 总结

整个仓库的核心创新在于：用**代理网络替代真实模型推理**来高效近似 Shapley 值，将 Shapley 计算的复杂度从 O(2^N) 降到分层采样 + MLP 前向推理的线性级别。流程清晰：采样数据 → 训练代理 → 估计贡献 → 删层 → 评估/微调。
